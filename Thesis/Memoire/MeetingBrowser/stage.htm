Title: Automatic true-false question answering in meetings

** Aim of the project:

Archives of meeting recordings are useful to check or to retrieve information from past meetings. However, such achieves are usable only if appropriate tools, generally named meeting browsers, are available to locate the information that is searched for. One approach to meeting browsing is to design general-purpose browsers that help human users to locate relevant information, but another possibility is to design browsers that locate information automatically, for instance for verification (fact checking) purposes.

The goal of this internship is to design such an automatic browser, and to assess its performance on a set of pairs of true-false statements, which have been initially used to evaluate human-directed browsers.

In other words, the goal is to design and to implement a system that determines the true and the false statement in each pair, to evaluate its performance over a set of several hundreds such pairs (for 4 recorded meetings), and to compare it with human subjects using existing meeting browsers. A comparative analysis of the system and the human scores on specific questions should indicate whether or not system and humans have the same difficulties answering such questions. This work will thus provide a baseline score against which human-directed browsers must be compared, to demonstrate significant improvement over a fully automated tool.

This internship requires competencies in programming for text manipulation and retrieval tools (e.g. in Java, Perl, C++), the ability to analyze statistical data from experiments, and an interest for new and open problems.
